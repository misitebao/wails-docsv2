关于

向 Go 应用程序提供界面的传统方法是通过内置的 Web 服务器。 Wails 提供了一种不同的方法：它提供了将 Go 代码和 Web 前端应用打包到一个二进制应用中的方法。 Wails Cli 可以帮您轻松实现应用创建，编译和打包。

  • 使用标准 Web 技术构建前端
  • 使用标准 Go 构建后端
  • 创建桌面或者服务器应用程序
  • 使用相同的代码编译到 Mac Windows 和 Linux (交叉编辑)
  • 一个强大的运行时库
  • 使用自动生成的绑定从 Javascript 调用 Go 方法
  • 支持平台原生功能，例如半透明 (MacOS), 无窗口 and 暗黑模式

Wails 附带的强大工具，可以简化创建和构建应用程序的过程。使用许多内置模板中的任何一个可以快速构建您的项目，然后生成平台原生应用程序。

  • 模板包含： Svelte、 Vue、 React、 Angular 和 Vanilla Javascript
  • 轻松创建平台原生应用程序 MacOS .app、 Windows .exe、 Linux Flatpack
  • 项目自动IDE配置 (VSCode)
  • 实验性支持，可为您的 Go 代码生成类型定义

Wails v2 已经被彻底重写，解决了我们在v1种遇到的许多问题。它具有许多新功能，包括：

  • 在 所有 平台都使用 Webkit 的自定义窗口库!
  • 应用菜单, 系统托盘, 右键菜单 和 原生对话框

开始使用

Wails v2目前正在MacOS上测试，Linux和Windows即将推出。

MacOS

安装说明：
  1. 确保您已经安装 Go v1.15 或者更高版本，并且 
    $PATH
    已经包含 Go 二进制文件。
  2. 确保您已安装最新版本的Node / npm 。
  3. 确保您已安装XCode Cli工具 
    xcode-select --install
  4. 克隆 Wails 仓库： 
    git clone https://github.com/wailsapp/wails
  5. 选择 
    feature/v2-mac
    分支
  6. 安装 v2 cli: 
    cd v2/cmd/wails && go install
笔记：
  • 目前不支持交叉编译
  • 某些模板尚未移植，目前仅支持 Svelte 和 Vanilla JS

创建你的应用

这些说明将创建一个基本的Svelte应用程序。看一下 
wails init --help
的其他选项。
  1. 为你的项目创建一个新目录并且 
    cd
    进入目录。
  2. 运行： 
    wails init -t svelte-mui -n test
    创建项目
  3. 运行： 
    wails build
    打包构建你的项目
  4. 应用程序将在 
    ./build/darwin/desktop/
    中构建
  5. 使用以下命令运行该程序： 
    ./build/darwin/desktop/test
